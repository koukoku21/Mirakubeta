# Miraku — Контекст проекта для Claude Code

## Что такое Miraku
Miraku — мобильное приложение, "Tinder для бьюти-услуг".
Соединяет клиентов с независимыми мастерами красоты (маникюр, стрижка, макияж и др.)
Запускается в Астане, Казахстан. 2026 год.

Главная механика: свайп-лента мастеров как в Tinder →
просмотр профиля → запись за 3 клика.

---

## Стек технологий

### Мобильное приложение
- Flutter 3.x + Dart
- Состояние: Riverpod 2
- Навигация: GoRouter
- HTTP: Dio + Retrofit
- Одно приложение для клиентов И мастеров (как InDrive)

### Backend API
- Node.js 20 LTS
- NestJS (фреймворк) + TypeScript
- ORM: Prisma
- База данных: PostgreSQL + PostGIS (геозапросы)
- Кэш: Redis (OTP коды, кэш слотов, WebSocket Pub/Sub)
- Auth: JWT (access + refresh) + SMS OTP через Mobizon KZ
- Чат: WebSocket (Socket.io) + Redis Pub/Sub
- Файлы: Cloudflare R2 (фото портфолио)
- Пуши: Firebase FCM

### Инфраструктура
- Хостинг: Railway (API + PostgreSQL + Redis)
- CDN: Cloudflare
- Монорепо: GitHub (apps/mobile + apps/api)
- CI/CD: GitHub Actions

### CRM для салонов (будущее, фундамент уже в схеме)
- Next.js 14 + Tailwind + shadcn
- Тот же NestJS API (отдельные модули)
- Хостинг: Vercel

## Архитектура базы данных (Prisma)

Полная схема в файле: schema.prisma

### Ключевые модели:
- **User** — один аккаунт, две роли (клиент + мастер)
- **MasterProfile** — профиль мастера, координаты (PostGIS), статус верификации
- **Service** — услуги мастера (цена, длительность, категория)
- **Schedule** — недельный шаблон расписания
- **ScheduleOverride** — особые дни
- **Booking** — записи (priceSnapshot фиксирует цену на момент записи)
- **BookingCommission** — сплит комиссий (платформа / мастер / салон)
- **Review** — отзывы только после COMPLETED записи
- **ChatRoom + ChatMessage** — чат (только с избранными мастерами)
- **Favourite** — избранные мастера (открывает доступ к чату)
- **SalonProfile + SalonMaster** — CRM фундамент (пустой на старте)
- **Notification + PushToken** — пуш-уведомления

### Важные правила БД:
- Soft delete через deletedAt (данные не удаляются физически)
- priceSnapshot в Booking — цена фиксируется в момент записи
- PostGIS для геофильтрации ленты мастеров
- OTP коды хранятся в Redis (TTL 5 мин), НЕ в PostgreSQL

## Все экраны приложения (35 экранов MVP)

### Auth (общий для всех) — 5 экранов
- A-1: Сплэш / Приветствие (без выбора роли)
- A-2: Ввод номера телефона (+7 KZ)
- A-3: Ввод SMS OTP кода
- A-4: Ввод имени (только новые пользователи)
- A-5: Запрос геолокации с объяснением

### Клиент — 16 экранов
- C-1: Лента мастеров (свайп-карточки)
  - Три кнопки: ✕ Пропустить | ♡ Избранное | 📅 Записаться
  - Свайп влево = пропустить, тап на карточку = профиль
- C-1a: Фильтр (боттом-шит): категория, радиус, цена
- C-2: Профиль мастера (портфолио, услуги, отзывы, адрес)
- C-2a: Портфолио полноэкранный просмотр
- C-2b: Все отзывы мастера
- C-3: Выбор услуги (шаг 1/3 записи)
- C-4: Выбор даты и времени (шаг 2/3, слоты с учётом буфера)
- C-5: Подтверждение записи (шаг 3/3)
- C-6: Список записей клиента (предстоящие / завершённые)
- C-6a: Детали записи
- C-7: Оставить отзыв (боттом-шит, 1-5 звёзд + текст)
- C-8: Профиль клиента + кнопка "Стать мастером"
- C-9: Список избранных мастеров
- C-9a: Пустое избранное
- C-10: Список чатов клиента
- C-11: Диалог с мастером (WebSocket)

### Tab Bar клиента (5 табов):
🏠 Лента | ♡ Избранное | 💬 Чаты | 📅 Записи | 👤 Профиль

### Мастер — 14 экранов
Онбординг мастера запускается из C-8 (профиль клиента):
- M-1: Специализации (теги-чипы)
- M-2: Адрес работы (2GIS autocomplete)
- M-3: Загрузка портфолио (мин. 3 фото)
- M-4: Добавить первую услугу
- M-5: Ожидание верификации (до 24 часов)

Основные экраны мастера:
- M-6: Дашборд (тогл "Принимаю", следующая запись, доход)
- M-7: Список записей мастера
- M-8: Расписание (недельный шаблон, дефолт Пн-Пт 10-19)
- M-8a: Особые дни (заблокировать дату/час)
- M-9: Редактирование профиля
- M-10: Портфолио (drag & drop сортировка)
- M-11: Управление услугами
- M-12: Настройки (переключение режим клиент/мастер, язык, выход)
- M-13: Чаты мастера (иконка в хедере экрана M-7)

### Tab Bar мастера (4 таба):
📊 Главная | 📅 Записи | 🕐 Расписание | 👤 Профиль

## Ключевая бизнес-логика

### Расчёт слотов (самая важная логика)
```
Свободные слоты = Рабочие часы мастера
                - Существующие записи (с учётом длительности услуги)
                - Буфер после каждой записи
                - Заблокированные часы и особые дни
```
Результат кэшируется в Redis (TTL 5 мин).
Инвалидируется при создании/отмене записи.

### Геофильтрация ленты
```sql
SELECT * FROM master_profiles
WHERE ST_DWithin(location, ST_Point($lng, $lat)::geography, $radius)
  AND is_active = true AND is_verified = true
ORDER BY ST_Distance(location, ST_Point($lng, $lat)::geography) ASC
LIMIT 20 OFFSET $offset;
```
Координаты мастера — адрес РАБОЧЕГО места (не GPS в реальном времени).
Мастер вводит адрес один раз через 2GIS autocomplete.

### Локация мастера
- Мастер указывает адрес рабочего места при регистрации
- Сохраняем lat/lng в БД (через 2GIS API при поиске адреса)
- PostGIS geography колонка добавляется raw SQL миграцией:
  `ALTER TABLE master_profiles ADD COLUMN location geography(Point,4326)`
- Геолокация мастера в реальном времени НЕ нужна никогда

### Чат
- Чат доступен только с мастерами из Избранного
- ChatRoom создаётся при добавлении в избранное
- WebSocket соединение устанавливается при открытии чата
- Сообщения хранятся в PostgreSQL
- Доставка через Redis Pub/Sub (для масштабирования)

### Auth флоу
- Один аккаунт для клиента и мастера (как InDrive)
- Регистрация всегда как клиент
- "Стать мастером" — в профиле клиента, запускает онбординг
- После одобрения в профиле появляется переключатель режимов
- Один JWT токен содержит userId и список ролей

### Верификация мастеров
- Мастер заполняет профиль → статус PENDING
- Менеджер смотрит в admin-панели → APPROVED или REJECTED
- До одобрения мастер не виден в ленте клиентов
- Admin-панель: простой Next.js, таблица заявок + две кнопки

## Важные решения которые уже приняты

1. **Один вход для всех** — нет выбора роли на A-1. Все входят как клиент.
2. **"Стать мастером" — в профиле клиента** (C-8), запускает онбординг M-1..M-5
3. **Координаты мастера — статические** (адрес работы), не GPS в реальном времени
4. **Чат только с избранными** — антиспам механика, ChatRoom создаётся при ♡
5. **priceSnapshot** — цена фиксируется в момент записи, не меняется
6. **Soft delete** — данные не удаляются физически (важно для CRM отчётности)
7. **Верификация мастеров** — ручная через admin-панель, до 24 часов
8. **Дефолтное расписание** — Пн-Пт 10:00-19:00, мастер меняет только что нужно
9. **Буфер 15 мин** — по умолчанию, мастер меняет в настройках расписания
10. **Слоты на 14 дней** — клиент видит расписание на 2 недели вперёд

## Чего НЕ делаем в MVP

- Встроенные платежи (клиент платит мастеру наличными или Kaspi QR)
- CRM дашборд для салонов (только фундамент в БД)
- Публичные страницы мастеров miraku.kz/@имя
- AI-рекомендации
- Магазин расходников
- Несколько адресов у одного мастера
